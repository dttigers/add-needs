**Status:** ✅ SHIPPED 2026-03-19

## Overview

Build a mobile app that intercepts ADHD cravings and translates them into real underlying needs, fast enough to act on before the impulse wins. Starting with project foundation and app shell, then the core redirect flow, followed by feedback and data persistence, pattern learning, and finally customization and polish.

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- Stack navigation (no tabs) — linear craving → result flow
- headerShown: false — custom UI control
- TypeScript strict mode from start — 0 errors always
- Zero async on mount — instant launch
- Module-level SQLite table init — no setup calls needed
- Sync SQLite only (runSync/getAllSync) — no async/await
- Hooks before guards — respect Rules of Hooks
- useFocusEffect on all data screens — reliable refresh
- Custom emoji fallback chain: CRAVINGS → getCustomMappings() → '?'
- Dark/light theme via system preference

**Issues Resolved:**
- MMKV v3 API change (createMMKV vs new MMKV)
- TypeScript strict DimensionValue cast for percentage bar widths
- Expo directory names with spaces workaround
- DATA-05 gap closed via Phase 6 (adaptive suggestions)
- Cross-screen data refresh via useFocusEffect (Phase 7)
- Custom emoji not displaying in history/stats (Phase 7)
- Dead MMKV module removed (Phase 7)

- None — all identified issues resolved
