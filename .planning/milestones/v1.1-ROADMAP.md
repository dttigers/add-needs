# Milestone v1.1: Suggestion Selection + Quick Launch Widget

**Status:** ✅ SHIPPED 2026-03-24
**Phases:** 8-9
**Total Plans:** 2

## Overview

Better feedback loop via tappable suggestions, then reduce friction to zero with an iOS widget. Phase 8 replaced the old yes/no/skip feedback with per-suggestion selection. Phase 9 added a WidgetKit home screen and lock screen widget that deep-links directly to the craving selection screen.

## Phases

### Phase 8: Suggestion Selection

**Goal**: Make suggestions tappable on result screen so users indicate which suggestion they'll try, plus an "Other" option — richer feedback data per craving
**Depends on**: v1.0 complete
**Plans**: 1 plan

Plans:
- [x] 08-01: Tappable suggestions + storage update

**Details:**
- Added chosen_suggestion column to redirect_events table
- Suggestions on result screen are now tappable cards — each tap records which specific suggestion the user chose
- Added "Other / none of these" option for cases where neither suggestion fits
- Removed old "Did that help?" yes/no/skip section — suggestion selection replaces it
- Added getSuggestionStats() query enabling future per-craving suggestion reordering
- Tapping a suggestion automatically sets feedback='yes'; "Other" maps to feedback='other'

### Phase 9: iOS Widget

**Goal**: Build and configure an Expo-compatible iOS widget (lock screen + home screen) that deep-links to the app's craving selection screen
**Depends on**: Phase 8 complete
**Plans**: 1 plan

Plans:
- [x] 09-01: iOS Quick Launch Widget

**Details:**
- Widget target scaffolded with SwiftUI code for two families: .systemSmall (home screen) and .accessoryCircular (lock screen)
- @bacons/apple-targets plugin integrated into Expo config
- Native iOS project generated via expo prebuild with QuickLaunch widget extension properly linked
- Deep-link via adhdneeds:// URL scheme to craving selection
- Static timeline with .never refresh policy — widget is just a launcher

## Milestone Summary

**Key Decisions:**
- Tapping a suggestion sets feedback='yes' automatically — no separate feedback step
- "Other/none" sets feedback='other' — distinct from skip (no interaction)
- No App Groups needed — widget is launch-only, no shared data
- Two widget families only (.systemSmall + .accessoryCircular) — minimal surface
- Static timeline with .never refresh — no dynamic content needed
- Skipped local xcodebuild verification due to hardware constraints

**Issues Resolved:**
- CocoaPods UTF-8 encoding error — resolved by setting LANG=en_US.UTF-8
- node_modules missing on session start — resolved with npm install

**Issues Deferred:**
- Visual widget verification — pending first run on capable device or EAS build
- Apple Team ID configuration — user must set in app.json before device build

**Technical Debt Incurred:**
- Widget visual verification not yet confirmed on device
- appleTeamId placeholder in app.json needs real value before distribution
