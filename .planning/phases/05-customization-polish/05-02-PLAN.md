  <action>
Create `app/manage.tsx` following the visual style of `history.tsx` (dark background, styled list, back navigation):

**Structure:**
- `backgroundColor: '#0a0a0a'`, `flex: 1`
- `StatusBar hidden`
- Header row: back button (`router.back()`) at top-left, title "Custom Mappings" centered
- FlatList of custom mappings from `getCustomMappings()`
- Each row shows: emoji + label (left), delete button (right, red ✕)
- Tap on a row (not on delete) navigates to `add-mapping` with params: `{ id: String(item.id), label: item.label, emoji: item.emoji, need: item.need, suggestion1: item.suggestion1, suggestion2: item.suggestion2 }`
- Delete button calls `deleteCustomMapping(item.id)` then refreshes list
- Empty state: centered text "No custom mappings yet" in `#666666`
- "+" Add button: absolute position `bottom: 32, right: 24` — navigates to `/add-mapping` with no params (new mapping mode)
- Refresh pattern: use `useState` to hold the custom mappings list, refresh by calling `getCustomMappings()` again after add/delete. Use `useFocusEffect` from `expo-router` to re-fetch on screen focus (so returning from add-mapping reflects newly saved data):
  ```ts
  import { useFocusEffect } from 'expo-router';
  import { useCallback, useState } from 'react';

  const [mappings, setMappings] = useState<CustomMapping[]>([]);
  useFocusEffect(useCallback(() => {
    setMappings(getCustomMappings());
  }, []));
  ```

<task type="auto">
  <name>Task 2: Create Add/Edit form screen</name>
  <files>app/add-mapping.tsx</files>
  <action>
Create `app/add-mapping.tsx` — a form that handles both Add (no id param) and Edit (id param present) modes:

**Route params:** `{ id?: string; label?: string; emoji?: string; need?: string; suggestion1?: string; suggestion2?: string }`

**Structure:**
- `useLocalSearchParams` to read params
- `isEdit = !!id` — drives title ("Add Mapping" vs "Edit Mapping") and save action
- Five `TextInput` fields with labels:
  1. Emoji — single character, `maxLength: 2`, `fontSize: 32`, centered (narrow input)
  2. Craving label — e.g., "Comfort snacking"
  3. Real need — e.g., "Emotional soothing"
  4. Suggestion 1 — e.g., "Hold something warm"
  5. Suggestion 2 — e.g., "Write down what you're feeling"
- Pre-fill inputs from params when in edit mode (use `useState` initialized from params)
- Validation: all 5 fields must be non-empty; show inline error text in `#ff6b6b` if Save attempted with empty fields
- Save button: calls `addCustomMapping(...)` or `updateCustomMapping(Number(id), ...)`, then `router.back()`
- Cancel/Back button: `router.back()` with no save

**Styling:**
- `backgroundColor: '#0a0a0a'`, full screen, `ScrollView` to handle keyboard
- Header: back button (←) top-left, title top-center
- Labels in `#666666`, `fontSize: 12`, `textTransform: 'uppercase'`
- TextInput style: `backgroundColor: '#1a1a1a'`, `borderRadius: 12`, `padding: 14`, `color: '#ffffff'`, `fontSize: 16`
- Save button: `backgroundColor: '#6366f1'`, full width, `borderRadius: 12`, `paddingVertical: 16`
- Use `KeyboardAvoidingView` wrapping `ScrollView` (behavior: `'padding'` on iOS) to prevent keyboard covering inputs
  </action>

  </done>
