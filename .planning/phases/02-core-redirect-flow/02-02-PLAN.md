Replace app/result.tsx with the full result display screen. The screen receives `cravingId` as a route param.

**Param reading:**
```typescript
import { useLocalSearchParams, router } from 'expo-router';
const { cravingId } = useLocalSearchParams<{ cravingId: string }>();
```

**Lookup:**
```typescript
import { CRAVINGS, MAPPINGS } from '../src/data/mappings';
const craving = CRAVINGS.find(c => c.id === cravingId);
const result = cravingId ? MAPPINGS[cravingId] : undefined;
```

**Fallback:** If cravingId is missing or not found (defensive — shouldn't happen in normal flow), show a simple "Something went wrong" message with a "Go back" button.

**Layout (full-screen dark, StatusBar hidden):**

Top section (flex: 1, justify center, align center, paddingHorizontal: 24):
- Small label: "Your craving" in muted gray (#666, fontSize: 13, fontWeight: '500', uppercase, letterSpacing: 1)
- Craving display: "{craving.emoji} {craving.label}" (fontSize: 20, color: '#888', marginTop: 4)
- Spacer / arrow or divider (a simple "↓" in indigo #6366f1, fontSize: 24, marginVertical: 20)
- Label: "Real need" in muted gray (#666, same style as above)
- Need name: result.need (fontSize: 32, fontWeight: '700', color: '#ffffff', textAlign: 'center', marginTop: 4)

Middle section (suggestions):
- Section label: "Try this" in muted gray (#666, fontSize: 13, uppercase, marginTop: 32, marginBottom: 12)
- Two suggestion cards side by side OR stacked (stacked is simpler — full-width cards):
  - Each card: backgroundColor: '#1a1a1a', borderRadius: 12, padding: 16, marginBottom: 10
  - Suggestion text: fontSize: 16, color: '#cccccc', lineHeight: 22
  - Optional bullet or number prefix ("1. " / "2. ") in indigo (#6366f1)

Bottom section:
- "Done" button (full-width, backgroundColor: '#6366f1', same style as Phase 1 CTA)
