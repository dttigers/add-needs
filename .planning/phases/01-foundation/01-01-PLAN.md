  <action>
Run from the project root (directory already exists):
```
npx create-expo-app@latest . --template default
```
This scaffolds an Expo Router + TypeScript project. After generation:

1. Update app.json: set `name` to "ADHD Needs", `slug` to "adhd-needs", `scheme` to "adhdneeds"
2. Enable TypeScript strict mode: add `"strict": true` under `compilerOptions` in tsconfig.json
3. Delete the `app/(tabs)/` directory — it's example boilerplate for a tabs layout we don't need
4. Delete `components/` directory if it contains only boilerplate examples (Collapsible, ThemedText, etc.)
5. Run `npx expo start --clear` to verify no startup errors, then quit (Ctrl+C)

Do NOT run `npm install` separately — create-expo-app handles it.

Create a simple Stack-based navigation — no tabs, just a linear craving → result flow.

app/_layout.tsx — Root layout with Stack navigator, no headers (we'll add custom UI later):
```tsx
import { Stack } from 'expo-router';

export default function RootLayout() {
  return (
    <Stack screenOptions={{ headerShown: false }}>
      <Stack.Screen name="index" />
      <Stack.Screen name="result" />
    </Stack>
  );
}
```
