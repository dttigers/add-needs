  <files>src/storage/events.ts, src/storage/index.ts</files>
  <action>
Create `src/storage/events.ts`:

1. Import `db` from `./sqlite`

2. Define and export the `RedirectEvent` TypeScript interface:
   ```
   export interface RedirectEvent {
     id: number;
     craving_id: string;
     craving_label: string;
     need_name: string;
     timestamp: number;       // Unix ms (Date.now())
     feedback: 'yes' | 'no' | 'skip' | null;
   }
   ```

3. At module level (not inside a function — runs once on first import, auto-initializing the table):
   ```
   db.runSync(
     `CREATE TABLE IF NOT EXISTS redirect_events (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       craving_id TEXT NOT NULL,
       craving_label TEXT NOT NULL,
       need_name TEXT NOT NULL,
       timestamp INTEGER NOT NULL,
       feedback TEXT
     )`
   );
   ```

4. Export `logEvent(cravingId: string, cravingLabel: string, needName: string): number`
   - INSERT a row with the given values, timestamp = Date.now(), feedback = NULL
   - Get the inserted id from `result.lastInsertRowId` (db.runSync returns SQLiteRunResult)
   - Return the id as a number

5. Export `setFeedback(eventId: number, feedback: 'yes' | 'no' | 'skip'): void`
   - UPDATE redirect_events SET feedback = ? WHERE id = ?
   - Use db.runSync with [feedback, eventId]

6. Export `getHistory(): RedirectEvent[]`
   - SELECT * FROM redirect_events ORDER BY timestamp DESC
   - Use db.getAllSync<RedirectEvent>(...)
   - Return the result array

Update `src/storage/index.ts` to add `export * from './events';` so events are accessible from the unified storage barrel.

Do NOT use async/await — expo-sqlite sync API only (db.runSync, db.getAllSync). TypeScript strict mode; no `any`.
